# 域专属测试清单(domains/)

按"被测域"拆分的专属漏测清单。通用分层在上一级 `../test-design-checklist.md`(§1-13,**任何域都过**);
这里放各域**额外**要逐项对的坑。

## 已沉淀的域
- `wallet.md` — 钱包:充值 / 提现 / 归集 / 资产 / 链特性(POL/TON/SOL/EVM)

## 约定
首次测到一个**新域**(如 `futures` 合约、`spot` 现货、`risk` 风控、`wallet-sign` 签名、或非加密业务),
按 `wallet.md` 的格式**新建 `<域>.md`**,把该域踩到/想到的坑沉淀进来。
这样技能**越用越全、不靠人肉喂**——这正是当初钱包域反复漏测后得到的教训。
